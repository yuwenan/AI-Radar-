# AI Radar · 资讯雷达

自动盯住全网 AI 动态，用大模型打分、翻译、去重，每天整理成一份中文日报推送到飞书群——你只管看结论。

还内置一套**变更监听**：盯住厂商官方 SDK 包（如 Claude Code / OpenAI / Gemini 等），在官宣之前就从包代码里 diff 出**新模型 ID、新工具、新发布通道**等早期信号，第一时间推送。

> 一个可自部署的 AI 资讯聚合 + 早期信号雷达：前端是配置后台，后端是「抓取 → 打分 → 翻译 → 去重 → 推送」的全自动流水线。

---

## 它能做什么

- **多平台数据源**：抖音 / 小红书 / B 站 / YouTube / X(Twitter) / GitHub / 任意 RSS / 任意网页，统一在「数据源」页里增删管理，开关随心。
- **AI 自动处理**：抓回来的内容先由大模型打一个「推送价值分」（相关性 + 热度），再翻译成中文，最后做链接去重 + 语义去重，干净入库。
- **每日飞书日报**：按「大厂动态 / 名人大事 / 国外 / 国内 / 博主观点 / GitHub 热榜」分区拼成卡片，定时推送；支持多个机器人分发到不同群。
- **变更监听（官宣前信号）**：定时抓取官方 SDK 包并 diff，捞出还没正式宣布的新模型 / 新能力，实时推送，并跨数据源做语义去重避免重复。
- **聊天式操作**：内置 AI 助手，用大白话就能查动态、手动跑一次日报、加 / 停某个数据源、调整过滤词——不用懂后台界面。
- **高度可配置**：关键词词典（屏蔽词 / 加权词）、推送时间与阈值、卡片模板、模型路由、可对接第三方打分工作流（Coze / Dify / n8n）。

## 配置后台一览

| 页面 | 作用 |
|---|---|
| 数据源 | 增删改各平台 / RSS / 网页源，分类、开关、抓取参数 |
| 资讯条目 | 抓回来的内容库，看分数、查重、人工干预 |
| 关键词 | 屏蔽 / 加权词典，控制什么该上、什么该压 |
| 飞书机器人 | 配机器人、分组推送、日报分发 |
| 推送模板 | 自定义日报卡片的版式与分区 |
| 模型路由 | 给翻译 / 打分 / 助手分别指定模型 |
| 工作流 | 对接外部打分 / 处理工作流 |
| 历史 / 日志 | 推送历史、抓取与任务日志，排障用 |
| 搜索 / 设置 | 全库检索、全局参数 |

## 它是怎么工作的

```
数据源(各平台 / RSS / 网页)
        │  定时抓取
        ▼
   大模型打分（相关性 + 热度）
        │
   翻译成中文 → 去重（URL + 语义）
        │
       入库
        │  按分区组卡片，定时
        ▼
   飞书机器人推送日报 / 信号
```

## 架构

```
前端  Next.js (App Router)  ──HTTP──>  后端 FastAPI + APScheduler  ──>  数据库 (SQLite / MySQL)
配置后台                                抓取 · 打分 · 去重 · 推送           │
                                        ├─ 大模型 API（翻译 / 摘要 / 打分 / 助手）
                                        ├─ RSSHub（推特 / B站 / 抖音等路由，自部署）
                                        └─ 飞书机器人（日报 / 信号卡片）
```

## 快速开始

### 后端（FastAPI）

```bash
cd backend
cp .env.example .env          # 按需填入大模型 API key、飞书 webhook 等
uv sync                       # 或：python -m venv .venv && pip install -e .
uv run uvicorn app.main:app --host 0.0.0.0 --port 3002
```

默认用 SQLite，开箱即跑，首次启动自动建表并塞一批默认数据源。生产环境可在 `.env` 把 `DATABASE_URL` 换成 MySQL。

### 前端（Next.js）

```bash
npm install
echo "NEXT_PUBLIC_API_BASE=http://localhost:3002" > .env.local
npm run dev                   # 开发：http://localhost:3000
# 生产：npm run build && npm start
```

## 依赖的外部服务（都可选、都走环境变量）

- **大模型 API**：翻译 / 摘要 / 打分 / 助手。没配也能起，只是内容处理会降级。
- **RSSHub**（自部署）：抓推特 / B站 / 抖音等需要。
- **飞书自定义机器人**：要推送就在「飞书机器人」页填 webhook。
- **抖音 / 小红书商业 API**：作为抓取兜底，可选。

所有凭证一律走环境变量，模板见 `backend/.env.example`。**请勿把真实密钥写进代码或提交进仓库。**
